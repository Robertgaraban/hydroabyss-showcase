# Public Scope

## Intent

This document defines what belongs in the public showcase repository and what should remain in the private production repository.

## Safe to publish

- public README and project narrative
- architecture summaries
- screenshots
- selected static pages or small public-facing samples
- design notes
- feature summaries
- sanitized examples

## Keep private

- deployment scripts
- infra details
- admin code
- contact processing internals
- raw source material
- internal prompts, agent instructions, and operating procedures
- marketing operations and outreach workflows
- analytics baselines and internal KPI tracking

## Publishing rule

If a file helps a third party understand the quality of the work, it may belong in the showcase.

If a file helps a third party reproduce your operations, deployment, or business workflow, it should stay private.

## Review-on-request rule

Full source access is available only on request and only when there is a real evaluation context.
